# Ollama + Qdrant Simple Demo
This project ingests sample text data into Qdrant using embeddings generated by Ollama, then lets you run semantic search, traverse all stored points, or chat with a tiny RAG endpoint backed by an Ollama generation model.

## Project files
- `app.py`: CLI with `ingest`, `query`, `traverse`, and `serve` commands
- `sample_data.json`: sample documents
- `requirements.txt`: Python dependencies

## Prerequisites
1. Qdrant running locally on `http://localhost:6333`
2. Ollama running locally on `http://localhost:11434`
3. Embedding model pulled in Ollama (default: `nomic-embed-text`)
4. Generation model pulled in Ollama for `serve` (default: `llama3.2`)

Example setup:
```bash
docker run --rm -p 6333:6333 qdrant/qdrant
ollama pull nomic-embed-text
ollama pull llama3.2
```

## Install
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Ingest sample data
```bash
python app.py ingest
```

This recreates the collection `ollama_demo_docs` and writes all sample docs.

## Query semantically
```bash
python app.py query --query "How do I run qdrant locally?"
```

You can set `--limit` to control the number of results.

## Traverse stored points
```bash
python app.py traverse --batch-size 3
```

`traverse` uses Qdrant scroll pagination so you can iterate through points regardless of similarity.

## Run the RAG chat endpoint
After ingesting documents, start the API server:
```bash
python app.py serve
```

By default this listens on `http://127.0.0.1:8000` and exposes:
- `GET /health` — liveness probe
- `POST /chat` — body `{"message": "...", "limit": 3}` returns `{"answer", "citations", ...}`

Example request:
```bash
curl -s http://127.0.0.1:8000/chat \
  -H 'Content-Type: application/json' \
  -d '{"message": "How do I run Qdrant locally?"}'
```

Useful flags: `--chat-model`, `--retrieval-limit`, `--host`, `--port`, `--collection`.

## Optional environment variables
- `QDRANT_URL` (default: `http://localhost:6333`)
- `OLLAMA_URL` (default: `http://localhost:11434`)
- `OLLAMA_MODEL` (default: `nomic-embed-text`) — embedding model
- `OLLAMA_CHAT_MODEL` (default: `llama3.2`) — generation model used by `serve`
